# 075 -- z3950: pure-Go Z39.50 client -> codex.Record (future version)

New capability. Tier: high effort, structural. Companion to the SRU client [074].
Ref: ANSI/NISO Z39.50 / ISO 23950; bib-1 attribute set; YAZ (reference impl, do
NOT port). Implement from the published standard.

## Motivation

SRU [074] reaches catalogs that speak the modern HTTP protocol, but a large number
of library targets -- legacy OPACs, and some LC/OCLC endpoints -- are **Z39.50
only**. Z39.50 is the same use case and returns the same record schemas as SRU
(MARC21/ISO 2709, MARCXML, SUTRS), so a Z39.50 client extends copy-cataloging reach
to those targets while feeding the exact same decode pipeline. Only the transport
differs: raw TCP + BER/ASN.1 + a stateful session, versus SRU's HTTP + XML.

## Shared shape with SRU (design constraint)

The two clients must converge on one abstraction: **transport -> record bytes ->
decode**. Concretely:

- Present the same `Reader` / `codex.RecordReader` surface as `sru`, so a Z39.50
  search is also a `codex.Convert` source and callers switch transport without
  changing downstream code.
- Reuse (don't duplicate) the record-schema normalization and the per-record MARC
  decode dispatch. Factor the shared bit out of `sru` -- either an unexported
  helper both packages call, or a small `internal/` package if it grows. MARC21
  binary records go through `iso2709.Decode`; MARCXML through `marcxml.Decode`.

## Scope

1. Pure-Go client implemented **from the Z39.50 standard, not a port of YAZ**
   (avoids license entanglement and the ~90% of YAZ -- servers, SRW, ZOOM, GRS-1,
   PQF/CQL -- that is not needed).
2. Session: `Init` (version/options/preferred-message-size negotiation), `Search`
   (Type-1/RPN query, result set), `Present` (fetch records in a chosen syntax),
   `Close`. `Scan` (index browse) later.
3. Query: build a Type-1/RPN query over the **bib-1 attribute set** (use, relation,
   position, structure, truncation, completeness). A small builder mapping common
   access points (title/author/isbn/issn/any) to bib-1 use attributes; do not
   attempt full generality first.
4. Record syntaxes: MARC21/USMARC (BER-wrapped ISO 2709) -> `iso2709.Decode`; also
   MARCXML and SUTRS. Return `*codex.Record` via the shared `Reader`.
5. Present/Reader paging: request records in batches, advancing the result-set
   cursor, ending on the result-set size.

## Hazards

- **BER/ASN.1 is the hard part.** `encoding/asn1` is DER-oriented and does not cover
  the Z39.50 APDUs cleanly (CHOICE types, implicit/explicit context tagging,
  indefinite lengths). Expect to hand-roll a small BER reader/writer for the Z39.50
  PDU set. Scope tightly: land Init/Search/Present with MARC21 first, one syntax.
- Stateful session over raw TCP: connection lifecycle, timeouts (via
  `context.Context`), preferred/maximum message size, and result-set naming.
- Character-set / record-syntax negotiation (Init `otherInfo`, `preferredRecordSyntax`).
- bib-1 attribute mapping is a table with real subtlety (relation/structure combos);
  get the common cases right, document the rest as gaps.
- No live network in tests: capture real BER PDUs (e.g. via a `yaz-client` session
  against a public target) into `testdata/` fixtures and replay them through a
  loopback `net.Pipe`/fake conn; assert Init/Search/Present decode + MARC records.
- License: implement from the standard; do not translate YAZ (or any GPL/BSD C)
  source line-by-line.

## Acceptance

- [ ] Init + Search + Present against a captured/fixture session returns MARC21
      records decoded to `*codex.Record`.
- [ ] Exposes the same `Reader` / `codex.RecordReader` surface as `sru`; the
      schema-normalization + decode dispatch is shared, not duplicated.
- [ ] bib-1 query builder for the common access points; RPN Type-1 query encoded.
- [ ] Tests replay captured BER fixtures (no live network); suite green.
- [ ] Deferred pieces (Scan, extra record syntaxes, full bib-1 coverage, servers)
      enumerated here as a checklist.
