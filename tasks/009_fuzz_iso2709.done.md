# 009 ‚Äî Thorough fuzz pass: iso2709

## Goal
Harden the `iso2709` binary codec with a thorough fuzz pass.

## Scope
- `FuzzDecode`: arbitrary bytes never panic; a successful decode re-encodes
  without panic (extends the existing `FuzzParseRecord`).
- `FuzzRoundTrip`: for inputs that decode cleanly, `Decode ‚Üí Encode ‚Üí Decode` is
  stable; assert structural equality on the second cycle.
- Seed corpus: valid records, truncated leaders/bodies, bad base addresses,
  out-of-range directory entries, oversized fields, MARC-8 byte sequences, and
  empty input.
- Sustained campaign: `go test -run=^$ -fuzz=Fuzz -fuzztime=‚Ä¶`; commit any
  crashers as regression seeds under `iso2709/testdata/fuzz`.

## Status ‚Äî done
- `FuzzDecode`: arbitrary bytes never panic; and because Encode normalizes the
  leader, the invariant is **Encode idempotence** ‚Äî `Decode‚ÜíEncode‚ÜíDecode‚ÜíEncode`
  is byte-stable. Campaign: **23.3M execs, 0 crashers**.
- `FuzzEncodeRoundTrip`: builds a record from fuzzed control/indicator/code/value
  data, then checks `Encode‚ÜíDecode` preserves the field data exactly (leader
  length/base are recomputed, so fields are compared). Campaign: **14.3M execs,
  0 crashers**.
- Seed corpus enriched: UTF-8 record, MARC-8 record, non-default entry map
  (4600), truncated record, bad base address, "short", empty.

## Finding fixed
The fuzz surfaced a real corruption vector: a value, indicator or subfield code
containing a reserved structural delimiter (`0x1d`/`0x1e`/`0x1f`) would corrupt
the record on read. **`Encode` now rejects such records** (`checkDelimiters`)
instead of silently corrupting; covered by `TestEncodeRejectsDelimiters`.

(The fuzz also corrected the test invariant itself: a naive `Decode‚ÜíEncode‚ÜíDecode`
DeepEqual fails because Encode legitimately normalizes the leader ‚Äî byte 9 ‚Üí 'a',
entry map ‚Üí "4500", length/base recomputed. The idempotence/field-equality
invariants above are the correct ones.)

## Acceptance ‚Äî met
- No panics or hangs over ~38M executions across two targets; idempotence and
  field round-trip invariants hold; iso2709 coverage 98.0%. No crashers to commit.
